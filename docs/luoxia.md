# 落霞宗 · 内容规格

> **落霞世界包唯一真相源**（剧情/信息差/钩子）。引擎分层见 `architecture.md`。

---

## 0. 核心设计原则（先对齐）

1. **只有种子**：背景、人设、性格、开局隐患 flags。开局时世界**暂停**在此——**没有**写好的后续剧本。  
2. **故事共写**：所有情节由 **NPC 心智 + 天道导演** 在运行时书写，不是内容包日历线。  
3. **线索 = 条件硬钩子**（可写死）：例如「首次与某人对话」「首次抵达某地」→ **固定派发一包**同构事件（events/ops）。  
   - 记在 `world_flags.fired_clues`，once。  
   - **禁止**台词关键词念咒；**禁止**洛晴阶段表/林溯动摇表等假进度条。  
4. **世界规则可自转**：倒计时、封山等是规则压力，不是分镜。  
5. **天道**：导演+裁决，产出 events；终局 settlement 收束。  
6. **UI** 只投影已有状态/履历；不渲染程序员结局标签。

---

## 1. 世界底层锚点（开局种子，非「禁改结局」）

> **锚点 = 仿真起点与信息差基准**，不是「玩家永远改不了」。  
> 玩家通过线索事件流，可以改写因果与立场；只是不能靠念一句台词就绕过事件包直接改权威。

### 1.1 三层真相（信息差基准）

| 层 | 内容 |
|---|---|
| **公开共识** | 玄阴殿来函称秘境将开、全宗筹备仪式（对外话术；与劫数倒计时并行，**不**等于日历剧情表）。无掌门；表面由**沈监察**与**云烨**共治。 |
| **半公开异常** | 护山阵偶发异动；人心浮动、谣言；高层议事频而无结论。 |
| **核心隐秘** | ① 密信为**假**；② 护山阵阵眼已种**血阴咒**，开启仪式≈引爆；③ **林溯**为玄阴殿内应；④ 至宝**落霞剑髓**由落云子临终交**洛晴**藏匿；⑤ 落云子坐化实为**玄阴殿下毒**所致。 |

> ④⑤ 写入开局种子 flags / 信念，并严格信息差（见 §2）。向玩家/多数 NPC **露出**时，必须走对应线索事件包。

### 1.2 核心运转规则

**双轨时间**

- **自然日**：每日行动点有限；`max_days` 与劫数初始 **21** 对齐。  
- **劫数倒计时**：初始 **21**（`xuanyin_countdown`）；默认日终 -1；玩家行为经事件包可加速/减缓。

**信念传导**

- 玩家对某 NPC 说出的信息，按该 NPC **性格 / 立场 / 社交圈**进入传谣与信念。  
- 锚定 **RumorPass**（同地、延迟、跳数、失真）+ 天道/心智写入的 belief_ops。  
- **倾向扩散**，禁止「固定 N 天全图同步」。

**权限与解锁**

- 锁闭区域、涉密信息：**无固定解锁日**。  
- 触发只认：相关线索事件包已落地 → flags / 信念 / 通告 / 引荐等硬状态。  
- 禁止单独维护「人情点数」。

**世界自转**

- 无为：倒计时走完 → 仪式/引爆路径可致覆灭。  
- 有为：假信、血阴、内鬼、托付等均可被事件流改写；终局只展示状态与履历。

## 3. 人物 · 行为倾向（AI 基准）

每人只保留：动机、默认态、正/负向倾向、秘密钩子。  
**不限制**台词、出场顺序；**默认活动区**仅为倾向，开局聚拢见 §8.3。

### 3.1 云烨（大师兄）

- **动机**：守住落霞基业，查清异常根源。  
- **默认态**：寡言；庶务与宗主殿一带；与人保持距离；暗查「有内鬼」，不锁定具体人。  
- **正向**：玩家给有效线索、护宗立场清晰 → 渐信，可共享进展，愿为玩家**引荐/担保**进入执法相关、藏经许可等。  
- **负向**：帮林溯开脱、干扰调查 → 提防、限制活动、监视。  
- **钩子**：持有落云子去世当日异常记录；相关硬状态满足后可主动出示——由对话意图 → 天道事件包触发，非日程、非数值条。

### 3.2 林溯（二师兄）

- **动机**：借玄阴扶持争位，压过云烨。  
- **默认态**：广场、任务堂；广结善缘；推进仪式叙事；掩盖密信破绽。  
- **正向**：玩家有利用价值、站秘境说法 → 示好、差事、便利、「内部消息」。  
- **负向**：查密信、戳破绽 → 先安抚试探，再使绊（谣言、扣便利、黑手）。  
- **钩子**：不知血阴会炸宗；**不知**落云子为玄阴下毒；得知炸宗真意图后可**动摇**（`lin_su_*` 线索事件包）。权威：`allegiance=xuanyin`，可 `exposed`。

### 3.3 洛晴（小师妹）

- **动机**：守遗命、护落霞剑髓、查师父死因。  
- **默认态**：**倾向**后山静修；公共区**低频率**；开局须保留**偶遇可能**，禁止做成「整月只存在于传闻、玩家无处触发」。  
- **正向**：持续善意、不出卖、分享异常线索 → 回避→提醒→托付→联手。  
- **负向**：硬探至宝、站林溯、泄露其行踪 → 封闭、深藏。  
- **钩子**：剑髓 + 遗命；托付后可全盘（分阶段信念/flags，经事件包推进，非一键倾倒）。  
- **实现**：沿用 `trust_player` / `trusts_player` / arc flags，不另造全局信任条。

### 3.4 沈监察（执法长老）

- **动机**：法度；无实证不动，有实证雷霆。  
- **默认态**：执法堂；通告发声；疑密信但缺锤。  
- **正向**：合规实证、按程序报案 → 立案、通告、开放相关权限。  
- **负向**：无据造谣、违规 → 禁足、盘查、取消客卿便利。  
- **钩子**：弟子档案；立案后可调林溯异常出入。唯一 `can_proclaim` 权柄脸。通告 = 意图 → 事件包，不扫「通告」二字台词。

### 3.5 苏婉（三师姐）

- **动机**：正统秩序、程序正义。  
- **默认态**：巡礼；零容忍混乱；服沈之法，不服云烨独断。  
- **正向**：守规矩、助其维持秩序 → 程序助力、殿外相关便利。  
- **负向**：违规传谣 → 呵斥、上报。  
- **钩子**：秩序崩塌时可走「秩序反噬」倾向（由事件包与状态体现）。

### 3.6 赵浦（任务堂）

- **动机**：机缘里捞利益；情报人情皆筹码。  
- **默认态**：任务堂；散播秘境话术；差事换人情。  
- **正/负**：给甜头↔断财路。  
- **钩子**：经手密信抄录；细问可见**格式漏洞**（假信线入口之一）。

### 3.7 明镜（藏经阁）

- **动机**：守典，认牌不认人。  
- **默认态**：藏经阁；无许可拒入。  
- **正/负**：正规许可/补残卷↔硬闯偷书。  
- **钩子**：旧录含血阴记载、剑髓来历、下毒旁证残片。

### 3.8 刘大壮（伙房）

- **动机**：安稳开饭（可保留「攒钱」等生活钩，不强制）。  
- **默认态**：伙房；八卦节点；听风是雨。  
- **正/负**：给甜头聊八卦↔得罪则抹黑。  
- **钩子**：见过林溯深夜往山门一带（传谣级，非实锤）。

### 3.9 韩铁（执法司事）

- **动机**：保职；可有把柄类生活压力（赌债等）作**可选钩子**，非必走勒索线。  
- **默认态**：随沈办事。  
- **钩子**：经手出入/笔录；可漏林溯记录（条件触发）。

### 3.10 岳山（山门）

- **动机**：守门；可有家人牵挂作可选钩。  
- **默认态**：认令盘查。  
- **钩子**：见过定期来寻林溯的外来人（传谣/目击级）。

### 3.11 白问舟（客司）

- **动机**：对接客卿，求稳求升。  
- **默认态**：客气、少涉阴谋。  
- **钩子**：客卿与外来登记；开局引导口（门径线索事件包）。

## 4. 三条主线 · 倾向与钩子（无顺序、非必走）

推进 = **条件线索包落地** 或 **天道/NPC 共写** → 信念 + flags + 事件卡。

### 4.1 假信线

**阶段倾向**：风闻 → 疑窦 → 坐实 → 公开  

| 跃迁 | 示例条件（经事件包落地） |
|---|---|
| → 疑窦 | 矛盾说法；密信格式/传讯路径异常（赵浦等）→ 写入调查 flags / 信念 |
| → 坐实 | 书证或人证 → `letter_exposed` 等（如线索 `letter_exposed`） |
| → 公开 | 沈通告意图事件包；或传谣仿真使大范围信念已不信秘境 |

### 4.2 血阴线

**阶段倾向**：异兆 → 查阵 → 破阵 / 爆发  

| 跃迁 | 示例条件（经事件包落地） |
|---|---|
| → 查阵 | 亲历异动 / 调查意图 → `investigating_curse` 等（如线索 `investigate_curse`）；**禁止**靠台词关键词「血阴」念咒 |
| → 破阵 | 阵图认知 + 解法 + 联手执行者（状态组合）。**不强制**动用落霞剑髓；剑髓主在人心线。洛晴同盟且持髓完成镇压 → 可选标签「剑髓安宗」 |
| → 爆发 | 倒计时归零；或仪式被强行启动 |

标志位：`blood_curse_*`、`sect_stabilized`、`crisis_*`；`jian_sui_required_for_disarm=False`。

### 4.3 人心线

**阶段倾向**：隔心 → 托重 → 同盟 / 反目  

| 跃迁 | 示例条件（经事件包落地） |
|---|---|
| → 托重 | 洛晴相关 flags 累积；无出卖/硬探至宝 |
| → 同盟 | 共享核心线索、愿共担风险（意图 → 如 `ally_shi_mei`） |
| → 反目 | 泄露剑髓；明确站林溯害她 |

至宝定名**落霞剑髓**、下毒真相进入托付后的信念——由事件包写入，非 UI 直接塞。

## 5. 动态钩子库（条件触发，非日程表）

禁止「第 N 天必刷事件」。下列均为**条件满足时可由内容钩子 / 天道抛出的框架**；对白自由，**结果落地必须成事件包**。

### 5.1 氛围向（高频、轻）

- 同地多人闲聊 → 可插话改局部信念，经传谣外溢。  
- 沈通告 → 大范围信念波动。  
- 伙房传谣 → 可纠正 / 加码 / 歪曲。

### 5.2 线索向（中频）

- 护山异动：证据类 flag 或倒计时偏低等条件。  
- 林溯拉拢 / 云烨试探 / 洛晴偶遇：条件钩子——**保证可触发**，非固定日程。

### 5.3 临界向（低频）

- **封山聚拢**：倒计时很低或假信半公开等 → 反向锁定、NPC 向核心区聚（§8.2）。  
- **内鬼揭发**：`exposed` 等实锤 → 立场分化。  
- **血阴爆发**：倒计时归零或仪式启动 → 终局结算。

## 6. 终局（天道结算，无硬编码结局线）

- **禁止**程序员/内容包 `if flags` 打结局标签；**禁止**在天道 system prompt 写具体剧情教条（下毒/假信/解咒等专有词表）。  
- 终局调用天道 `phase=settlement`：输入 player 快照、履历 chronicle、world_flags、结束 reason → 产出 `narrative_summary` + 终局 events。  
- 终局 UI：结束原因 + **天道收束文** + 玩家权威状态 + 本局事件履历。

## 7. 与实现的映射（必须遵守）

| 规格概念 | 落地处 |
|---|---|
| 线索 / 意图 | 天道同构事件包；条件线索走 `on_dialogue` / `on_move` |
| 倒计时 21 + `max_days` | `WORLD_FLAGS` / `WorldRules` |
| 硬状态 | flags + beliefs（事件包**之后**） |
| UI 投影 | `clue_projection`：见闻 / 隐情灰字；履历看 events |
| 假信/血阴/内鬼/动摇 | `content/luoxia/clues.py` 等事件包 + 夜演/危机 tick |
| 剑髓、下毒种子 | 开局 flags；向玩家露出走事件包 |
| 林溯不知会炸 / 不知下毒 | 开局信息差；动摇走 `lin_su_*` 事件包 |
| 剑髓非破阵门槛 | `jian_sui_required_for_disarm=False` |
| 封山阈值 | sticky `seal_mountain`；`countdown≤5` 或（`letter_exposed` 且 `≤10`） |
| 传谣 | `RumorPass`（跳数失真，无词替换念咒表） |
| 解锁地图 | 线索事件包 → `map_unlocked` |
| **禁止** | 台词关键词驱动推进；日历剧情表；Graph 写死分镜 |

## 8. 开局开放、地图锁与线索展示（条件制，非日历）

### 8.1 开局体验

高自由 ≠ 开局空洞：

- 开局须能在**生活圈 + 广场**碰到足够主线脸。  
- 洛晴须存在「可触发的偶遇」。  
- 须知只给公开共识 + 行动方式，不剧透核心隐秘。  
- **禁止**「第 N 天自动开门 / 自动给线索」。

### 8.2 地图：起始开放集 + 条件解锁

| 集合 | 地点（建议） | 如何变化 |
|---|---|---|
| **开局即开放** | 外门客居、伙房、山门、落霞广场 | 起始状态 |
| **条件解锁** | 执法堂、任务堂、藏经、宗主殿、内门、后山、演武、长老院等 | 线索事件包落地（引荐/通告/flags）——**无固定天数** |
| **临界反向锁** | 封山时外门/山门等 | **`xuanyin_countdown ≤ 5`**，或 **`letter_exposed` 且 `countdown ≤ 10`** |

### 8.3 开局人物聚拢（默认落点倾向）

| 角色 | 开局倾向落点 |
|---|---|
| 玩家、白问舟 | 外门客居 |
| 刘大壮 | 伙房 |
| 岳山 | 山门 |
| 林溯、赵浦、云烨、苏婉 | 广场一带 |
| 沈、韩铁 | 执法堂（锁则靠偶过广场/通告感） |
| 洛晴 | 后山倾向 + **条件偶遇**公共区 |
| 明镜 | 藏经（锁内） |

日终移动仍由意图批次/倾向决定，不按日程赶人。

### 8.4 线索：事件包 + 投影（勿再写反）

1. **发线索** = 天道发出同构事件包（见 §0.5）。  
2. **硬状态** = 事件包 apply 之后的 flags / beliefs / world_flags / events（真相字典，见 `architecture.md`）。  
3. **UI（一律投影，不发明第二套文案）**：  
   - **见闻**：`player.beliefs` ← `beliefs[player]`（`clue_projection.enrich_belief_row`）  
   - **隐情**：`clue_flags` ← `world_flags` + 知情判定（顶栏劫数也读此轨）  
   - **情境**：`player.situation_rows` ← `states[player]`  
   - **谈话流**：`chat_by_actor` ← `session.dialogue`（对白权威；正文展卷仍读 `events`）  

解锁地图、钩子、终局标签，一律读硬状态（= 事件流结果），不读「玩了第几天」。  
禁止：`narrative_summary` / `graph_meta.settlement_summary` / `last_effects` / 前端硬编码当玩家真相。终局展示用 `settlement_text`（事件投影）。

## 10. 战斗系统（独立专节 · 勿与对话/入夜/线索混写）

> 权威结算走遭遇意图 → ContentPacket / Applier；前端「画」面只加载表现模块。

### 10.1 机制：交锋制（短对峙）

- **结构**：三国志式单挑味——先排手，再对放；**不是**长回合 JRPG。  
- **一局**：通常 **1～3 次对放** 收束。  
- **气**：本场资源；**每场开局回满**。上限由境界定（§10.2）。  
- **功法**：背包 `kind=gongfa`；只提供 **封闭词条字母表**上的组合（皮可未知）。  
- **小招**：开战时按功法生成（**LLM 花式优先，校验失败 → 规则组合回退**）；战斗中不再调 LLM。  
- **操作**：点选并排顺序（Σ耗气 ≤ 气，至多 3 手）→ **己方确定** → 服务端为对方 **AI 排招** → 按序对拼。  
- **无联机**：本作为单人世界模拟；对方出手不由另一玩家点选，而由服务端按气与小招表自动排。  
- **结算**：词条修正 + 修为线性放大后比有效攻守；胜负/伤势只许服务端写回。  
- **分层**：词条定棋盘，修为定砝码。

```text
切磋/应战 → start（ContentPacket 写 active_encounter + 开战事件）
  → 生成双方 player_catalog / foe_catalog 写入 blob
点选 pick → 更新 player_moves
确定 confirm → AI 排 foe_moves → 对放结算 → 清 active_encounter + 结果事件
罢手 cancel → 清 active_encounter
```

### 10.2 词条 / 数值表（封闭骨 · 皮可无限）

> 生成器与引擎 **只认本表**；未知 id 在 `validate_move` 拒收。

**境界 → 气上限**

| 境界前缀 | 气上限 |
|----------|--------|
| 练气/炼气 | 5 |
| 筑基 | 7 |
| 结丹/金丹 | 9 |
| 元婴 | 11 |
| 化神 | 13 |
| 默认 | 5 |

**修为放大**：`1 + layer * 0.08`，只乘 `axes` 数值轴。

### 10.3 开战与 LLM 边界

| 何时 | 行为 |
|------|------|
| 人物 **切磋** / 应战条 | `encounter` `op=start` |
| 天道张力足够 | `events` + `ui_hints.propose_encounter` → `encounter_offer`（**不**直接开打） |
| 开战瞬间 | 可选 LLM 生成小招 → `sanitize_moves`；失败规则组合 |
| **禁止** | 台词关键词开打；`world_flag_ops.active_encounter` 直写 |
| 战斗中 | **不** 调 LLM 写规则；只用 blob 内 catalog |

### 10.4 Prompt 约定

天道 / 对话快路径须声明：

- 存在交锋（短对峙，气 + 小招）。  
- 约战/单挑 → **必须** `events`；可 `propose_encounter`。  
- 胜负不归叙事句；禁止 `active_encounter` 直写。  
- pack `dynamic_prompt_extra.combat` 注入字母表与字段说明。

### 10.5 功法与小招生成

**功法包**：`art_id` / `name` / `lore` / `tags` / `axis_bias` / 可选 `realm_min`。  
**默认功法**：客途吐纳诀（`art_guest_breath`）；开战若无则 `encounter_ensure_default_art_packet` 补发。  
**NPC 无功法**：宗门外门粗功底库。  

**生成路径**（`WorldPack.encounter_build_catalogs`）：

1. `duel_moves_llm.generate_moves_for_arts`（LLM）  
2. `duel_demo.sanitize_moves` 硬校验  
3. 不足 → `compose_moves_from_art` 规则组合  
4. 结果写入 `active_encounter.player_catalog` / `foe_catalog`

### 10.6 动作与 DTO

| op | 含义 |
|----|------|
| `start` | 开战 |
| `pick` | 更新己方招式序列 |
| `confirm` | 对拼结算 |
| `cancel` | 罢手清交锋 |
| `dismiss_offer` | 罢议应战要约 |

| DTO | 含义 |
|-----|------|
| `SessionView.encounter` | 进行中交锋（moves/气/picked…） |
| `SessionView.encounter_offer` | 天道要约（foe_id/reason） |

画模块：`duel`（`DuelPlayPanel`）。
